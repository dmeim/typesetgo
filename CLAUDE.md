# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Quick Reference

**TypeSetGo** is a React + Convex typing practice platform with real-time multiplayer ("Connect"), user stats/leaderboards, and themes.

## Commands

```bash
# Development (two terminals needed)
bun run convex:dev      # Start Convex backend
bun run dev             # Start Vite frontend (port 3000)

# Build & Test
bun run build           # TypeScript check + Vite build
bun run test:run        # Unit tests (single run)
bun run test            # Unit tests (watch mode)
bun run lint            # ESLint

# Deployment
bun run convex:deploy   # Deploy Convex functions
```

**Before committing:** Run `bun run build && bun run test:run` to catch type errors and regressions.

## Architecture

- **Frontend:** Vite SPA with React 19 + TypeScript (`src/`)
- **Backend:** Convex real-time functions (`convex/`)
- **Auth:** Clerk (optional, enabled when `VITE_CLERK_PUBLISHABLE_KEY` is set)
- **Styling:** Tailwind CSS v4 + Radix primitives + Shadcn patterns

**Import alias:** `@/` → `src/`

### Key Directories

| Path | Purpose |
|------|---------|
| `src/pages/` | Route-level pages |
| `src/components/ui/` | Shadcn/Radix UI primitives |
| `src/hooks/` | Custom React hooks |
| `src/lib/` | Utilities, schemas, stores |
| `convex/schema.ts` | Database schema + indexes |
| `public/themes/` | Theme JSON files (auto-manifested) |

### Auto-Generated Files (Do Not Edit)

- `convex/_generated/*` — Convex generates these
- `public/*/manifest.json` — Generated by `vite-plugin-auto-manifest.ts`

## Conventions

- Functional components only (no class components)
- No `"use client"` directives (this is Vite, not Next.js)
- Double quotes, semicolons, 2-space indentation
- Components: `PascalCase.tsx`, utilities: `kebab-case.ts`
- Prefer existing `@/components/ui/` primitives over new base components
